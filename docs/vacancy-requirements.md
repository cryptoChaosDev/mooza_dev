# Вакансии — тестируемый чеклист требований (из ТЗ)

Каждый пункт — проверяемое утверждение. Тестировщик отмечает PASS / FAIL / PARTIAL + доказательство
(file:line) + замечание. Проверять РЕАЛИЗАЦИЮ (код фронта/бэка + поведение), а не план.

Ключевые файлы: server/src/routes/vacancies.ts, server/src/routes/posts.ts, server/src/routes/artists.ts,
server/prisma/schema.prisma; client: components/VacancyForm.tsx, pages/VacancyDetailPage.tsx,
pages/VacanciesPage.tsx, pages/ArtistPage.tsx, pages/FeedPage.tsx, pages/FlowSettingsPage.tsx,
lib/api.ts (vacancyAPI), lib/vacancyOptions.ts, App.tsx.

## A. Создание, права, форма, поля (ТЗ 3, 4)
A1. Создавать вакансию может только Артист (тип Компания не реализован) — гейт по владению артистом (UserArtist isOwner+ACCEPTED ИЛИ Artist.submittedById=создатель).
A2. Откликаться может Резидент (Представитель компании не реализован) — архитектурно отклик доступен любому залогиненному не-автору.
A3. Точка входа 1 — раздел «Мои вакансии» в профиле артиста (кнопка «Добавить»).
A4. Точка входа 2 — Поток: «Создать пост» → тип «Вакансия» → та же форма (с выбором артиста, если артистов >1; если 1 — авто; если 0 — заглушка).
A5. Форма ОДНА И ТА ЖЕ независимо от точки входа.
A6. При публикации вакансия ОБЯЗАТЕЛЬНО появляется и в Потоке, и в «Мои вакансии».
A7. Раздел каталога = каталог ПРОФЕССИЙ (не услуг). Фильтры профессии подтягиваются по выбранной профессии.
A8. Поля формы (4.1) присутствуют и корректны:
   - Название вакансии (своб. ввод, обязательное)
   - Раздел каталога = профессия (обязательное)
   - Фильтры по профессии (по выбранной профессии, необязательное)
   - Формат работы (single: Онлайн/Офлайн/Гибрид, обязательное)
   - География (single: свой город/свой регион/по стране/международная, обязательное)
   - Тип занятости (single: постоянная/частичная/проектная/стажёр/волонтёр, обязательное)
   - Тип оплаты (single: бесплатно/за респект/бартер/процент/ставка, обязательное)
   - Размер вознаграждения (число, необязательное, ВИДЕН ТОЛЬКО при оплате «Процент»/«Ставка»)
   - Описание (многострочное, обязательное)
   - Материалы: файлы (картинки jpg/png/gif/webp + аудио mp3/wav/flac/ogg, суммарно ≤20МБ) + список ссылок («+ Добавить ссылку», источники Яндекс.Диск/Google Docs/Dropbox/YouTube), необязательное
   - Тоггл «Комментарий обязателен при отклике» (обязательная настройка)
   - Тоггл «Портфолио обязательно при отклике» (обязательная настройка)
A9. Кнопка внизу формы — «Опубликовать».
A10. Тихое автосохранение в черновик при незавершённом заполнении/закрытии формы (без подтверждения), как у услуги/заказа.

## B. Подходящие кандидаты и мэтчинг (ТЗ 3.3, 3.4, 14)
B1. На странице вакансии у автора СРАЗУ появляется живой список «Подходящие кандидаты».
B2. Мэтчинг по профессии + фильтрам профессии — ОБЯЗАТЕЛЬНОЕ условие попадания.
B3. Сортировка по статусу занятости: «Открыт»/«Рассматриваю» выше, «Закрыт» — в последнюю очередь (не скрывается); «Не указан» — показывается.
B4. На карточке кандидата виден его статус занятости (Не указан/Рассматриваю/Открыт/Закрыт).
B5. Кандидат кликабелен → переход в профиль.
B6. Рядом с кандидатом кнопка «Предложить вакансию» → кандидату приходит уведомление «Вам предложили вакансию» → переход на страницу вакансии.
B7. Пустое состояние (ТЗ 14): полностью пустого нет в обычном режиме; крайний случай (нет кандидатов с проф.) — заглушка с акцентом «вакансия опубликована в Потоке» + автообновление.

## C. Поток, видимость, «Мои вакансии», «Все вакансии», карточка (ТЗ 6, 7, 12, 13)
C1. Вакансия в Потоке = публикация типа «Вакансия» (новый тип в реестре типов публикаций).
C2. Карточка в Потоке (компактно): Название, Раздел (профессия), Формат работы, Условия (тип оплаты), География, краткое описание (первые символы + многоточие).
C3. На карточке в Потоке кнопка ТОЛЬКО «Посмотреть детали»; НЕТ «Написать» и «Откликнуться».
C4. Вакансию можно найти в Потоке через фильтр по типу.
C5. «Мои вакансии» видны только автору (как заказы/сделки).
C6. Блок «Мои вакансии» в профиле артиста: слайдер плиток; первая плитка «Добавить»; в заголовке счётчик активных + «Посмотреть все» → «Все вакансии».
C7. Компактная плитка в блоке: Название, Раздел (профессия), Формат работы.
C8. «Все вакансии»: три вкладки — Активные, В архиве, Черновики.
C9. Карточка в «Все вакансии»: Название, Профессия, Формат работы + кнопки по статусу: Активная → «Редактировать»/«В архив»; Черновик и В архиве → «Редактировать»/«Опубликовать».
C10. Страница вакансии (автор): все поля; кнопка «В архив»; блок «Подходящие кандидаты»; блок «Отклики».
C11. Пустое «Мои вакансии» — только плитка «Добавить».

## D. Отклик, оффер, статусы, переписка (ТЗ 5, 8, 9, 10, 11)
D1. Форма отклика открывается по «Откликнуться» на странице вакансии (доступна соискателю-не-автору).
D2. Поля отклика: Комментарий (обязательность зависит от настройки вакансии) + Портфолио (файлы+ссылки, обязательность зависит от настройки). Поля цены НЕТ.
D3. Если поле обязательно и не заполнено — отправка блокируется + уведомление.
D4. После отправки отклика автору приходит уведомление «[Имя] откликнулся на вакансию [название]»; отклик появляется в блоке «Отклики».
D5. Блок «Отклики» показывает всех откликнувшихся (независимо от канала); для каждого: имя, комментарий (если есть), портфолио (если есть).
D6. У каждого отклика кнопки «Написать» (→ переписка) и «Выбрать кандидата» (→ форма оффера о сотрудничестве).
D7. Форма оффера о сотрудничестве: Дата начала (обяз.), Описание условий (обяз.), Вознаграждение (обяз.), Доп. детали (необяз.). После отправки соискателю приходит уведомление.
D8. Можно отправить НЕСКОЛЬКО офферов по разным откликам одной вакансии.
D9. Соискатель видит оффер и может Принять/Отклонить (явной кнопки отказа от «Предложить вакансию» нет — только отклик/игнор).
D10. Принять → автору уведомление о принятии + системный вопрос «Архивировать вакансию?» (Да→архив, Нет→активна).
D11. Отклонить → автору уведомление; вакансия остаётся активной без доп. вопросов.
D12. Статусы: только режимы Активная/В архиве/Черновик (нет формальной статусной модели переходов); переключаются вручную; авто-архив ТОЛЬКО при принятии оффера (с вопросом); дедлайна нет.
D13. Переписка: «Написать» из блока «Отклики» И со страницы вакансии ДО отклика (соискатель может написать автору). Обычный диалог без авто-структурированного сообщения.
D14. Добавление принятого кандидата в участники — НЕ автоматизировано (вручную).
