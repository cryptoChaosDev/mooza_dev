# Moooza — Скоуп MVP
> Выделено из MOOOZA_WORK.md. Включает только то, что должно войти в первую версию продукта.

---

## Ценность MVP для пользователя

- **Профиль-визитка** музыканта / специалиста
- **Специалист** → получает клиентов и заказы
- **Заказчик** → находит специалистов и работает с ними
- **Причастность** к музыкальной тусовке

---

## Минимальный user flow

1. Пользователь регистрируется
2. Заполняет профиль (имя, профессия из каталога, локация)
3. Добавляет услуги
4. Добавляет портфолио
5. Ищет специалиста в каталоге
6. Открывает услугу → нажимает «Написать» или «Оформить сделку»
7. Переходит в чат → обсуждают детали
8. Оформляют сделку → оплата → работа → сдача → приёмка
9. Ставят оценки и оставляют отзывы
10. Выстраивают профессиональные связи
11. Размещают посты в Потоке
12. Добавляют друзей

---

## Типы пользователей в MVP

Только один тип профиля: **Резидент** (деятель музыкальной индустрии).

При выборе другого типа (Слушатель, Представитель компании, Заказчик) — форма сбора email для уведомления о запуске.

---

## 1. Регистрация

- Регистрация по email с подтверждением кодом (4 цифры, срок 15 минут)
- Регистрация через VK
- Чекбокс согласия на обработку ПД (обязательный)
- Чекбокс на рассылки (необязательный)
- Возрастное ограничение: регистрация с 16 лет; финансовые операции — с 18

---

## 2. Авторизация и вход

- Вход через VK
- Вход по email + пароль
- Восстановление пароля (код на email, срок 15 минут)
- Повторный вход без потери сессии

---

## 3. Стартовое заполнение профиля

### Карточка 1 — Имя, Фамилия, Ник, Аватарка
- Имя и Фамилия — обязательны
- Ник — необязателен, уникален (проверка в реальном времени)
- Если VK — данные подтягиваются, можно изменить

### Карточка 2 — Профессия (обязательна минимум одна)
- Поиск с ограничением по каталогу профессий (autocomplete)
- После выбора — раскрываются фильтры профессии (мультиселект)
- Можно добавить несколько профессий; до 10 профессий в профиле
- Кнопки: «Сохранить» / «Пропустить» (сохраняется только название без фильтров)

### Карточка 3 — Локация
- Страна и город — поиск с выпадающим списком по каталогу
- Подсказка: «Если не нашли свой город, выберите ближайший»

## 7. Профиль пользователя (свой)

**Шапка:** обложка (баннер), аватарка, ФИ, ник, локация, статусы (Верифицирован и др.), счётчики Связи/Сделки, кнопка шеринга, кнопка «Редактировать»

**Блоки:**
- **О себе** — если заполнен; если нет — блок отсутствует
- **Профессия** — кликабельный текст, при нажатии — попап с фильтрами
- **Артисты** — горизонтальный слайдер (3.5 плитки): логотип, название, роль. Первая плитка — «Добавить»
- **Услуги** — счётчик + слайдер активных услуг + «Смотреть все» → страница Услуги (вкладки: Действующие / Черновики / Архив)
- **Мои сделки** — счётчик + слайдер активных сделок + «Смотреть все»
- **Отзывы** — слайдер (3.5 плитки): тег типа, тег услуги, оценка, текст. «Смотреть все» → страница отзывов с сортировкой
- **Портфолио** — три раздела: Аудио / Изображения / Другое. В каждом — слайдер + кнопка «Добавить»
- **Социальные ссылки**

### Режим редактирования профиля
- Имя, Фамилия, Ник (проверка уникальности), Страна, Город, Дата рождения (маска 00.00.0000)
- О себе (многострочный ввод, 100 символов, счётчик)
- Профессии (до 10, поиск по каталогу, фильтры при выборе, кнопки «Редактировать» / «Удалить»)
- Соцсети и контакты
- Автосохранение при переходе между блоками
- Кнопка «Завершить редактирование»
- Кнопка «Превью» — посмотреть как другие видят профиль

## 8. Профиль другого пользователя

**Шапка:** обложка, аватарка, ФИ, ник, локация, статус сети (в сети / N минут назад), статусы, счётчики Связи/Сделки, кнопки действий

**Кнопки действий:**
- «Установить связь» — 1 тап = подсказка, 2 тап = действие
- «Добавить в друзья» / «Удалить из друзей»
- «Добавить в избранное»
- «Написать» → чат
- «Пожаловаться»

**Индикатор времени ответа:** «Обычно отвечает за N минут» (формируется системой; не показывается до первых взаимодействий)

**Блоки:** О себе, Профессия (с попапом фильтров), Артисты, Услуги, Отзывы, Портфолио, Связи (последние 3 + «Смотреть все»)

**Контакты и соцсети:**
- Незарегистрированный: заглушка «Зарегистрируйтесь, чтобы посмотреть»
- Зарегистрированный с незаполненным профилем: «Заполните профиль, чтобы посмотреть»
- Зарегистрированный с заполненным профилем: видит контакты

## 9. Каталог

### Раздел Услуги

**Структура:** три уровня — группы услуг → конкретные услуги → исполнители.

**Карточка услуги:** аватарка, имя, средняя оценка (если есть), название услуги, стоимость, прайс, кнопки «Написать» и «Оформить сделку».

**Фильтры:** специфичные для услуги из каталога + Локация (мультиселект) + Стоимость (от/до).

**Сортировка:** по дате / по стоимости / по оценке.

### Раздел Артисты

**Чипсы:** Соло-артист, Группа, Кавер-бэнд + жанры (комбинируются).

**Фильтр:** расширенная фильтрация по иконке.

**Сортировка:** по дате / по алфавиту.

**Карточка:** аватарка, название, тип, жанры, статус верификации.

### Раздел Люди

**Поиск:** по имени, фамилии, нику, профессии.

**Фильтры:** Локация (мультиселект), Профессия (мультиселект), Статус занятости.

**Сортировка:** по релевантности / по оценке / по количеству связей / по алфавиту.

**Карточка пользователя:** аватарка, ФИ, статусы, локация, профессии, количество связей. Раскрывается: О себе, профессии, оценка, кнопка «Перейти в профиль».

## 10. Статусы занятости

Три статуса (выбор один из трёх, radio button):

| Статус | Цвет |
|---|---|
| Закрыт | 🔴 Красный |
| Рассматриваю | 🟡 Жёлтый |
| Открыт | 🟢 Зелёный |

При «Рассматриваю» или «Открыт» — дополнительные мультиселекты: **География** (свой город / регион / страна / международная), **Тип занятости** (постоянная / частичная / проектная / стажёр / волонтёр), **Тип оплаты** (бесплатно / бартер / процент / ставка).

**Сетап оборудования** — мультиселект, зависит от профессии (гитарист, вокалист, барабанщик и т.д.).

**Синхронизация:** изменение статуса в профиле → предложение опубликовать апдейт в Поток.

## 11. Поток

**Типы публикаций в MVP (5 типов):**

### 1. Свободный блог
Текст, фото (до 10), теги, жанры, город, ссылки, упоминания.

### 2. Вопрос / обсуждение
Заголовок (обязательный), текст вопроса (обязательный), категория, фото, жанры, город, теги.

### 3. Опрос
Вопрос (обязательный), варианты ответов (обязательных), длительность. Голосование с процентами и счётчиком.

### 4. Апдейт услуги
Выбор опубликованной услуги + тип апдейта (Свободные места / Новый кейс / Обновление портфолио / Изменение доступности / Временное предложение / Другое) + текст. Ограничение: не чаще 1 раза в 24 часа на услугу.

### 5. Апдейт занятости
Новый статус занятости (radio) + текст. Только от имени физлица.

**Автор публикации:** можно выбрать — профиль физлица, канал физлица, канал артиста.

**Действия под постом:** лайк, комментарии, репост, поделиться, пожаловаться, сохранить (⭐).

**Навигация:** чипсы по типу публикации и типу автора. Сортировка: по новизне / по популярности / по релевантности.

**Лента для нового пользователя:** наполнена постами команды Moooza.

## 12. Сообщения

Три вкладки: **Личные | Деловые | Групповые**

### Личные чаты
Стандартный мессенджер (Telegram-подобный). Черновик сохраняется. Статус прочтения (1 галочка / 2 галочки). Поиск по чатам. Действия: закрепить, архивировать, удалить.

### Деловые чаты
Создаются **автоматически** при переходе сделки в «В работе». Вручную создать нельзя. Удалить нельзя — только в архив (при завершении или отмене сделки). Нельзя редактировать и удалять сообщения — полная история договорённостей.

### Групповые чаты
Создаёт любой пользователь. При добавлении без отношений — приглашение с кнопками «Вступить» / «Отказать». Администратор = создатель.

**Всем типам:** прикрепление файлов, реакции (emoji), закрепление сообщений, раздел вложений (Фото / Видео / Музыка / Файлы / Ссылки).

## 14. Отношения

- Поиск + кнопка «Онлайн»
- Баннер «Запросы дружбы» → страница запросов (Получено / Отправлено)
- В строке друга: кнопки «Написать» и «Удалить из друзей»

- Баннер «Запросы связи» → страница запросов (Получено / Отправлено)
- Список связей — переход на страницу связи

### Избранное
- Список профилей. Иконка сердечко — убрать из избранного. Уведомления при удалении нет.

## 16. Верификация артиста

1. Пользователь создаёт карточку артиста → получает уникальный код
2. Размещает код в описании профиля артиста в соцсетях (ВК, Instagram и др.)
3. Вставляет ссылку на профиль артиста → отправляет на верификацию
4. Саппорт проверяет → публикует карточку (или отказывает с причиной)
5. Пользователь становится администратором артиста

До верификации артист не отображается в каталоге.

## 17. Профиль Артиста

**Поля:** обложка, аватар, название (обязательно), тип (Соло / Группа / Кавер-бэнд), локация, жанры.

**Обязательные поля для создания:** название, аватарка, тип, мин. участников (Соло=1, Группа/Кавер=2), мин. 1 релиз (для Соло и Группы).

**Разделы страницы:**
- Участники (аватарка, ФИ, роль — кликабельны)
- Релизы (слайдер: обложка, название → страница релиза со ссылкой на стриминг и участниками)
- Клипы (слайдер: обложка, название → страница клипа со ссылкой на видеоплатформу и участниками)
- Описание, Соцсети, Контакты
- Административный блок (только для администраторов)

**Два пути добавления участника:** из зарегистрированных пользователей ИЛИ ссылка-приглашение для незарегистрированных.

| Поле | Тип | Обязательное |

| Раздел каталога | Autocomplete по каталогу | Да |

- **Действующие:** «Убрать в архив», «Оформить сделку»

- **Архив:** «Опубликовать»

### Страница у заказчика
- Кнопки «Написать» (шаблоны первого сообщения) и «Оформить сделку»

## 19. Сделки

### Тип А — Процессная (результат сдаётся по завершении)

**Статусы:** На согласовании → Ожидает оплаты → В работе → На проверке ↔ На доработке → Завершена / Отменена

**Поля:** название, услуга, фильтры, сумма заказа, срок сдачи (дни), срок приёмки (1–5 дней, default 3), результат, количество правок, исполнитель, заказчик.

### Тип Б — Событийная (оказывается в конкретную дату)

**Статусы:** На согласовании → Ожидает оплаты → Ожидает события → Ожидает подтверждения → Завершена / Отменена

**Поля:** название, услуга, фильтры, сумма заказа, депозит (невозвратный), дата события, результат, исполнитель, заказчик.

### Финансовая логика
- Комиссия платформы: 10%
- Гонорар исполнителя: 90%
- Безопасная сделка (холдирование) — через платёжный сервис
- Реквизиты исполнителя: токенизация карты (только пост-MVP)

### Страница сделки — кнопки (Тип А)

| Статус | Исполнитель | Заказчик |
|---|---|---|
| На согласовании | «Отменить» (если оформил) | «Принять», «Отклонить» |
| Ожидает оплаты | «Отменить» | «Оплатить», «Отменить» |
| В работе | «Сдать работу», «Отменить» | «Принять» (неакт.), «На доработку» (неакт.), «Отменить» |
| На проверке | «Сдать работу» (неакт.), «Отменить» | «Принять» (акт.), «На доработку» (акт.), «Отменить» |
| На доработке | «Сдать работу», «Отменить» | «Принять» (неакт.), «На доработку» (неакт.), «Отменить» |
| Завершена | «Оценить работу» | «Оценить работу» |
| Отменена | «Оценить работу» | «Оценить работу» |

**Кнопка «Редактировать»:** до принятия — все поля; после — только Срок сдачи, Срок приёмки, Количество правок (с согласованием второй стороной).

**Постоянные кнопки:** «Назад», «Написать» (→ деловой чат), «Дублировать».

**Автоматические переходы:**
- «В работе» → «Отменена»: если истёк срок сдачи (деньги возвращаются заказчику)
- «На проверке» → «Завершена»: если истёк срок приёмки (деньги выплачиваются исполнителю)

## 20. Оценка пользователя

**10-звёздочная шкала** (1 — Ужасно … 10 — Восхитительно).

**Два независимых канала:**
1. По сделке (кнопка «Оценить работу») — доступна при «Завершена» и «Отменена»
2. На странице вручную установленной связи — только если связь ручная (не автоматическая после сделки)

**После отправки:** оценка учитывается в среднем значении; пользователю приходит уведомление; появляются кнопки «Ответить» (для оцениваемого) и «Редактировать» (для автора).

---

## 21. Модерация контента (базовая в MVP)

- Автофильтр + постмодерация по риск-скору для большинства контента
- Обязательная **премодерация** для рекламных материалов
- Услуги: выборочная премодерация → статус `pending_review` перед публикацией
- Жалобы: на пользователя, на публикацию, на отзыв → в саппорт и админ-панель
- Блокировки: временная и постоянная по решению модератора

## Что НЕ входит в MVP

- ❌ Реферальная программа (первое обновление)
- ❌ PRO-аккаунт (первое обновление)
- ❌ Реальная оплата через платёжный сервис (пост-MVP)
- ❌ Токенизация карты исполнителя (пост-MVP)
- ❌ Типы публикаций: Событие, Вакансия, Коллаборация, Кейс, Продажа/Аренда, Релиз, Блог-статья, Апдейт проекта
- ❌ Аналитика (платный раздел)
- ❌ Кейсы на странице услуги
- ❌ Репутация (отдельно от оценки)
- ❌ Шифрование сообщений
- ❌ Настройки уведомлений (только базовые)
- ❌ Технические ошибки сервера / 404 (пост-MVP)
- ❌ Типы профилей: Слушатель, Представитель компании, Заказчик
- ❌ Типы Б (событийная) — блок финансов/депозита (требует платёжного сервиса)
